# casis.py

Скрипт для создания текстового снимка проекта, ориентированного на совместную работу с ИИ.  
Автоматически собирает **только файлы с числовыми индексами** (например, `script-01.py`, `ui_03.js`), выбирает последнюю версию в каждой группе и сохраняет всё в один файл — `project_for_ai.txt`.

## Зачем

При итеративной разработке с ИИ легко накопить много временных файлов.  
`casis.py` помогает:
- чётко отделить рабочие версии от черновиков,
- автоматически выбрать актуальные файлы,
- получить единый текст для передачи ИИ или архивации.

## Как использовать

### 1. Создать новый проект
```bash
python3 casis.py -n ./my_project
```
Создаст:
- `filters.txt` — список включаемых/исключаемых расширений и папок,
- `project_for_ai.txt` — шаблон снимка.

### 2. Работать с файлами
Именуйте файлы с числовым суффиксом:
```
core-01.py
core-02.py   ← будет выбрана эта версия
app_01.js
app_05.js    ← и эта
utils.py     ← игнорируется (нет индекса)
```

Поддерживаются форматы:
- `name-01.ext`
- `name_02.ext`
- `name03.ext` (если перед цифрами есть буквы)

Не поддерживаются: `01.py`, `final.py`, `draft-v2.py`.

### 3. Обновить снимок
```bash
cd my_project
python3 casis.py
```
Скрипт:
- соберёт все файлы с индексами,
- выберет последнюю версию в каждой группе,
- обновит содержимое между `***` в `project_for_ai.txt`,
- сохранит историю, комментарии и статистику вне маркеров.

## Фильтрация

Файл `filters.txt` определяет, какие расширения учитывать и какие папки/файлы игнорировать. Пример:
```txt
# ВКЛЮЧИТЬ
*.py
*.js
*.md

# ИСКЛЮЧИТЬ
.git
node_modules
*.log
.env
```
